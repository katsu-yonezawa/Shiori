# SHI-017 iPhone 版の実装方式を比較する

## 概要

| 項目 | 内容 |
| --- | --- |
| 優先度 | P2 |
| マイルストーン | M5: 将来展開の確認 |
| 依存 | SHI-001 |

## 目的

iPhone対応の方式を早い段階で見極め、将来の手戻りを減らします。

## 作業内容

- Tauri mobile と React Native / Expo を比較する
- ローカルDB、同期、エディタ、通知、配布方法の観点で整理する
- 既存の `packages` をどの程度再利用できるか確認する
- 初期版後に採用判断できる材料を作る

## 受け入れ条件

- 候補方式ごとの利点、制約、運用負荷が整理されている
- Shiori の要件に対する推奨方式が示されている
- 追加検証が必要な項目が分かる
