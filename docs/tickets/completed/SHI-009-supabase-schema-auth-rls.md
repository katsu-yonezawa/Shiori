# SHI-009 Supabase の認証、テーブル、RLS を設計する

## 概要

| 項目 | 内容 |
| --- | --- |
| 優先度 | P0 |
| マイルストーン | M3: 認証と同期 |
| 依存 | SHI-003 |

## 目的

クラウド同期に必要なユーザー単位の保存領域を安全に用意します。

## 作業内容

- Supabase Auth の利用方式を決める
- Postgres 側の `notes`、`tags`、`note_tags`、`sync_events` 相当のテーブルを設計する
- `user_id` を基準に Row Level Security を設定する
- ローカルSQLiteとの型差分を確認する

## 受け入れ条件

- ログイン中のユーザーが自分のデータだけ読める
- 他ユーザーのデータを読めないことをテストで確認できる
- ローカルDBとクラウドDBの対応関係がドキュメント化されている
