# SHI-011 ローカル変更を `sync_events` に記録する

## 概要

| 項目 | 内容 |
| --- | --- |
| 優先度 | P0 |
| マイルストーン | M3: 認証と同期 |
| 依存 | SHI-004, SHI-006 |

## 目的

差分同期のため、ローカル変更を後から送信できる形で記録します。

## 作業内容

- ノート作成、更新、削除のイベント形式を定義する
- タグ変更のイベント形式を定義する
- 送信前、送信中、送信済み、失敗の状態を管理する
- 同じノートへの連続更新をまとめる方針を決める

## 受け入れ条件

- ローカル変更ごとに同期イベントが記録される
- 送信済みイベントと未送信イベントを区別できる
- アプリ再起動後も未送信イベントが残る
