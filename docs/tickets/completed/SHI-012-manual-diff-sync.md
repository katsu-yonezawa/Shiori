# SHI-012 手動同期で push / pull できる差分同期を実装する

## 概要

| 項目 | 内容 |
| --- | --- |
| 優先度 | P0 |
| マイルストーン | M3: 認証と同期 |
| 依存 | SHI-009, SHI-010, SHI-011 |

## 目的

まずは利用者が明示的に実行できる同期を成立させます。

## 作業内容

- 未送信の `sync_events` を Supabase へ push する
- サーバー側の更新を pull してSQLiteへ反映する
- 最終同期時刻または同期カーソルを管理する
- 同期失敗時のリトライ方針を決める

## 受け入れ条件

- 手動同期で別環境の変更を取得できる
- ローカルの未送信変更をクラウドへ反映できる
- 同期失敗時にローカルの変更が失われない
- 同期ログまたはエラー内容を確認できる
