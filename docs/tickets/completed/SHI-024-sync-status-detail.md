# SHI-024 同期状態の詳細表示を改善する

## 概要

| 項目 | 内容 |
| --- | --- |
| 優先度 | P2 |
| マイルストーン | M4: 安全性向上 |
| 依存 | SHI-012, SHI-014 |

## 目的

同期が成功しているか、保留や失敗があるかを落ち着いて確認できるようにします。

## 作業内容

- 最終同期時刻、保留件数、失敗件数を分かりやすく表示する
- 同期失敗時に、ユーザーが次に取れる行動が分かる文言を表示する
- オフライン、未ログイン、Supabase未設定の状態を区別して見せる
- 必要に応じて、失敗した同期イベントの簡易一覧を確認できるようにする
- 同期状態の表示がノート編集の邪魔にならない配置にする

## 受け入れ条件

- 最終同期時刻、保留件数、失敗件数を確認できる
- 同期できない理由が状態別に分かる
- 失敗時に再同期を試せる
- 同期状態の表示が通常の編集操作を妨げない
- 既存の簡易自動同期と手動同期の動作を変えずに表示だけ改善されている
