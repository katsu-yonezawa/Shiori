# SHI-025 同期ボタンに頼らない自動同期体験を改善する

## 概要

| 項目 | 内容 |
| --- | --- |
| 優先度 | P1 |
| マイルストーン | M3: 認証と同期 |
| 依存 | SHI-012, SHI-013 |

## 目的

日常利用で同期ボタンを押す必要を減らし、ログイン中であれば自然にクラウドへ反映される状態にします。

## 作業内容

- 表示中かつオンライン時の定期同期を30秒間隔に調整する
- ノートの自動保存が成功したあと、短い待ち時間を置いて自動同期を実行する
- タグ変更、ノート作成、ノート削除など、本文以外の変更後も必要に応じて自動同期する
- 同期中に重複して同期が始まらないよう、既存の実行中ガードを維持する
- 未ログイン、オフライン、Supabase未設定のときは、過剰な通知を出さずに同期を見送る
- 手動の同期ボタンは、明示的な同期確認や失敗時の再試行として残す

## 受け入れ条件

- ログイン中かつオンライン時、表示中のアプリで30秒ごとに静かに同期される
- ノートの自動保存後、ユーザーが同期ボタンを押さなくても短時間で同期が試行される
- ノート作成、削除、タグ変更後も同期ボタンに頼らず同期が試行される
- 同期中に複数の同期処理が重複実行されない
- 未ログインやオフライン時に、通常の編集操作を妨げる通知が出ない
- 手動同期ボタンは、必要なときに再試行できる導線として機能する
