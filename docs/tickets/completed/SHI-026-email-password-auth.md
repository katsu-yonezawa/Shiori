# SHI-026 メール・パスワードログインを追加する

## 概要

| 項目 | 内容 |
| --- | --- |
| 優先度 | P1 |
| マイルストーン | M3: 認証と同期 |
| 依存 | SHI-010 |

## 目的

Magic Link のメール送信制限によりログインできなくなる状態を避けるため、メール・パスワードによるログイン手段を追加します。

既存の Magic Link は補助的なログイン方法として残しつつ、日常利用ではメール送信に依存せずログインできる状態を目指します。

## 背景

現在のデスクトップアプリは Supabase Auth の Magic Link のみを使っています。この方式では、初回ログインや再ログインのたびに認証メール送信が必要です。

Supabase の組み込みメール送信にはレート制限があり、開発中に複数回ログインを試すと `email rate limit exceeded` によりログインできない場合があります。セッション復元は実装済みですが、ログアウト後やセッション失効時の再ログイン手段が Magic Link に限られている点が課題です。

## 作業内容

- Supabase Auth のメール・パスワードログインを `auth.ts` に追加する
- 必要に応じてサインアップ、または既存ユーザーのパスワード設定方法を整理する
- ログインUIにメール・パスワード入力を追加し、Magic Link と切り替えられるようにする
- ログイン成功後のセッション保存と同期処理が既存実装と同じように動くことを確認する
- Magic Link のレート制限エラーが出た場合に、パスワードログインへ誘導できる文言を検討する
- Supabase 側で必要な Email provider 設定をドキュメント化する

## 受け入れ条件

- メール・パスワードでログインできる
- メール・パスワードログインでは Magic Link メール送信が発生しない
- ログイン後、既存の同期処理がログイン中ユーザーとして動作する
- アプリ再起動後にセッションが復元される
- Magic Link は従来どおり利用できる
- 未ログイン、認証失敗、レート制限時の表示が区別できる
- Supabase 側で必要な設定や注意点が分かる
